## Concept: Why Foundations Come First

Deep learning looks like model classes, GPUs, datasets, and giant APIs from the outside. Under the hood, it is mostly this:

```text
numbers -> tensors -> matrix operations -> gradients -> parameter updates
```

If you understand tensors and gradients, PyTorch stops feeling magical. A neural network becomes a set of numbers being adjusted by math.

```text
Raw data
  |
  v
Tensor shape: (batch, features)
  |
  v
Matrix multiply: X @ W + b
  |
  v
Prediction
  |
  v
Loss
  |
  v
Gradient
  |
  v
Update weights
```

That loop is the heart of deep learning. This first repo teaches every piece of that loop slowly.

## Concept: A Tensor Is a Number Container With Shape

```text
Scalar: 5
Shape: ()

Vector: [1, 2, 3]
Shape: (3,)

Matrix:
[[1, 2, 3],
 [4, 5, 6]]
Shape: (2, 3)

Image batch:
16 RGB images, each 224x224
Shape: (16, 3, 224, 224)
```

Human-English version:
> A tensor is a box of numbers. The shape tells you how the numbers are arranged.

Why this matters:
> Most PyTorch bugs are not "AI bugs." They are shape bugs.

### Task 15 - Human-English Breakdown: Broadcasting

Task 15 is the deep-dive task. Broadcasting is when PyTorch makes smaller tensors act like larger tensors so element-wise operations work.

The big idea:
> PyTorch compares shapes from right to left. Dimensions must match, or one of them must be 1, or one can be missing.

#### Lab 15a - Scalar to vector

```python
import torch
x = torch.tensor([1, 2, 3])
print(x + 10)
```

Human English:
> Add 10 to every value.

What we are proving:
> A scalar broadcasts to any shape.

#### Lab 15b - Vector to matrix

```python
import torch
M = torch.tensor([[1, 2, 3],
                  [4, 5, 6]])
v = torch.tensor([10, 20, 30])
print(M + v)
```

Human English:
> Add the same row to every row in the matrix.

Shape story:

```text
M: (2, 3)
v:    (3)
```

What we are proving:
> A `(3,)` vector can broadcast across the first dimension of a `(2, 3)` matrix.

#### Lab 15c - Column vector to matrix

```python
import torch
M = torch.zeros(2, 3)
col = torch.tensor([[100], [200]])
print(M + col)
```

Human English:
> Add 100 to row 0 and 200 to row 1.

What we are proving:
> A `(2, 1)` tensor can stretch across columns.

#### Lab 15d - Break broadcasting on purpose

```python
import torch
a = torch.zeros(3, 4)
b = torch.zeros(3)
print(a + b)
```

Human English:
> This fails because the rightmost dimensions are 4 and 3.

What we are proving:
> Broadcasting is not guessing. The rules must match.

#### Lab 15e - Fix it with `unsqueeze`

```python
import torch
a = torch.zeros(3, 4)
b = torch.zeros(3).unsqueeze(1)
print((a + b).shape)
```

Human English:
> Turn `(3,)` into `(3, 1)`, then stretch the 1 across 4 columns.

What we are proving:
> Most broadcast fixes are just one `unsqueeze` away.

#### Lab 15f - Use `None` as shorthand

```python
import torch
b = torch.tensor([1, 2, 3])
print(b[:, None].shape)
print(b[None, :].shape)
```

Human English:
> `None` inserts a dimension of size 1.

What we are proving:
> `b[:, None]` and `b.unsqueeze(1)` are two ways to express the same idea.

#### Lab 15g - Outer product with broadcasting

```python
import torch
x = torch.tensor([1, 2, 3])
y = torch.tensor([10, 20, 30, 40])
print(x[:, None] * y[None, :])
```

Human English:
> Make one tensor a column and the other a row, then multiply every pair.

What we are proving:
> Broadcasting can replace nested loops.

#### Lab 15h - Broadcasting in neural networks

```python
import torch
batch = torch.randn(32, 128)
bias = torch.randn(128)
print((batch + bias).shape)
```

Human English:
> Add the same bias vector to every sample in the batch.

What we are proving:
> Bias addition in dense layers is broadcasting.

#### Lab 15i - Broadcasting in attention masks

```python
import torch
mask = torch.tril(torch.ones(5, 5))[None, None, :, :]
print(mask.shape)
```

Human English:
> Convert a 2D causal mask into a shape that can broadcast across batch and heads.

What we are proving:
> Transformer attention uses the same broadcasting idea.

#### Lab 15j - Final mental model

```text
1. Write both shapes.
2. Right-align them.
3. Compare from the right.
4. Each pair must be equal, 1, or missing.
5. If a pair fails, use unsqueeze or reshape.
```

What we are proving:
> Broadcasting is readable once you turn it into a shape checklist.

### Task 20 - Scenario: image batch to model-ready tensor

**Purpose:** Simulate the shape pipeline used by real computer vision models.

```python
import torch

images = [
    torch.randint(0, 256, (224, 224, 3), dtype=torch.uint8)
    for _ in range(16)
]

batch = torch.stack(images)
print("HWC batch:", batch.shape)

batch = batch.permute(0, 3, 1, 2).contiguous()
print("CHW batch:", batch.shape)

batch = batch.float() / 255.0
device = "cuda" if torch.cuda.is_available() else "cpu"
batch = batch.to(device)

print("final:", batch.shape, batch.dtype, batch.device)
```

**Switches**

| Token | Meaning |
|---|---|
| `torch.randint` | Simulate image pixels from 0 to 255 |
| `torch.stack` | Combine 16 images into one batch |
| `.permute(0, 3, 1, 2)` | Convert `(B, H, W, C)` to `(B, C, H, W)` |
| `.float() / 255.0` | Convert pixel integers to normalized floats |
| `.to(device)` | Move to CPU or GPU |

**Output decoded**

| Output | Meaning |
|---|---|
| `(16, 224, 224, 3)` | Common image loading format |
| `(16, 3, 224, 224)` | PyTorch model input format |
| `float32` | Model-ready numeric type |

**Troubleshoot**

| Symptom | Fix |
|---|---|
| Model expects 4D input | Make sure you stacked images into a batch |
| Convolution complains about channels | Check whether you forgot `.permute(0, 3, 1, 2)` |

## Common Pitfalls

| Mistake | Symptom | Fix |
|---|---|---|
| Ignoring shape | Random PyTorch errors | Print `.shape` first |
| Confusing element-wise multiply with matrix multiply | Wrong math | Use `*` for element-wise and `@` for matrix multiplication |
| Forgetting batch dimension | Model expects 4D but gets 3D | Add a leading batch dimension |
| Forgetting channel order | Vision model gives channel error | Convert HWC to CHW |
| Mixing CPU and GPU tensors | Device mismatch error | Move everything to one device |
| Modifying a view accidentally | Original tensor changes | Use `.clone()` |
| Forgetting `requires_grad=True` | No gradients | Enable autograd on the tensor that needs gradients |

**For a future ML engineer**
> Practice narrating shapes aloud. Example: "X is batch by input features. W is input features by output features. The result is batch by output features." This is how strong engineers debug fast.

**For a future AI engineer**
> Transformers, embeddings, attention masks, and vector databases all rely on tensor shapes. This lab is the base layer for everything later.

**For teaching others**
> Keep the human-English explanation before the formal explanation. A beginner should understand the story before memorizing the API.
